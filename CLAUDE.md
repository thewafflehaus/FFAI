## Build / test / bench / profile

- Local loop: `make regenerate-kernels` then `make build`.
- **DO go through `make` for tests; DO NOT run bare `swift test`.** Each `Tests/ModelTests/*` suite loads multi-GB HuggingFace snapshots; parallel runs pin the GPU and OOM the box.
  - `make test-unit` — `FFAITests` + `MetalTileSwiftTests` (fast, parallel-safe).
  - `make test-integration` — `ModelTests`, serialized (`--num-workers 1`).
  - `make test` — both in sequence; the full local CI gate.
  - `make coverage` — unit-suite coverage. Line coverage SHOULD stay ≥ 80 %.
  - `make test-stress` — production cap, uncapped parallelism; DO run it after touching dispatch code.
- After `make regenerate-kernels`, DO run the empty-body MSL detector (documented in metaltile's `developing.md`) over `Sources/MetalTileSwift/Resources/kernels/*.metal`. A hit ships all-zeros output and is a release blocker.
- Longer-form testing story: [`documentation/developing/testing.md`](documentation/developing/testing.md).

### Porting kernels from our `ekryski/mlx` fork

Custom kernels (turbo / gated-delta / SSM / …) live on **`ekryski/mlx@alpha`** under `mlx/backend/metal/kernels/` (local checkout `~/Development/personal/ai/mlx`, branch `alpha`). metaltile's `MLX_COMMIT` is pinned there so `tile bench` has a side-by-side reference.

- DO try existing metaltile DSL primitives + `#[kernel]` macros first.
- Missing a primitive? DO extend the DSL (body parser → IR → MSL codegen). DO NOT hand-write raw MSL.
- New AURA / GDN / SSM / vision / audio kernels DO start under `crates/metaltile-std/src/ffai/`; graduate to `mlx/` once an MLX counterpart ships at the pinned commit.
- After each port, DO diff the metaltile-generated MSL against the `alpha` reference kernel.

### Wrapping a kernel in a Swift `Ops` function

An `Ops` wrapper is the boundary between caller-supplied dimensions and the GPU. For **reduction-mode kernels** (`simd_*` / `threadgroup_alloc` / `KernelMode::Reduction` in the `.rs`, or a fixed `tpg=` in the `BenchSpec`) it is the only thing enforcing the geometry contract the kernel can't check at runtime:

1. DO read the kernel's `## DISPATCH INVARIANTS` block. If it has none, DO add one in the same commit.
2. DO `precondition` every invariant at the top of the wrapper, before any allocation or dispatch. Cite the kernel `.rs` in the message.
3. DO derive dispatch geometry from the invariants. DO NOT use `elementwiseGrid` for a reduction kernel — that is the canonical freeze bug; it is for genuine one-thread-per-output Grid3D kernels only.
4. DO add a `Tests/FFAITests/OpsTests.swift` test exercising the wrapper at a production-realistic shape.

### Testing — FFAI side

DO ship a paired GPU correctness test in the same commit for every non-trivial `ffai/` or `mlx/` kernel (test layers: `docs/testing.md`). What FFAI tests in Swift:

- **Per-Op** — `Tests/FFAITests/Ops*Tests.swift`: every `Ops.<fn>` against a small asserted output. New Ops DO land with a test.
- **Per-cache** — `Tests/FFAITests/*StateCacheTests.swift`: append / slice / serialize round-trip + multi-layer working-buffer sharing.
- **Determinism** — `Tests/ModelTests/DeterminismSmokeTests.swift`: temp=0, same prompt, stable across runs.
- **Per-model coherence** — `Tests/ModelTests/<Family>IntegrationTests.swift`: load, greedy-decode, assert `expectCoherentOutput(...)`. DO NOT compare to mlx-lm / mlx-vlm goldens — per-kernel GPU correctness tests are the tighter signal; the contract is "the model produces coherent text".
- **Coverage** — `make coverage` ≥ 80 % line.

**Promotion path:** new kernels start under `metaltile-std/src/ffai/` with `BenchDispatch::Generic` + a paired GPU correctness test. Backport verified bench shapes into metaltile once confirmed; graduate to `mlx/` if an MLX counterpart ships at the pinned commit.

DO use these commands; full CLI reference in `docs/cli.md`. Run from each repo's root.
