## Swift / Apple platform

- Swift 6 strict concurrency. Use `Sendable`, `actor`, structured concurrency. Avoid `@unchecked Sendable` unless there's a defensible reason.
- Target the latest macOS / iOS SDKs. Use new APIs when an older fallback isn't strictly required.
- Prefer value types (`struct`, `enum`) over reference types unless identity / shared state is genuinely needed.
- `MTLBuffer` directly — no `MLXArray` middleware.
- `// swiftlint:disable` and `@available` annotations only with a one-line reason.

### CLI primer (`tile --help` for the full surface)

- `tile build --emit all --out <FFAI/Sources/MetalTileSwift>` — codegen MSL + metallib + Swift wrappers + manifest.
- `tile bench --filter <kernel>` — MLX side-by-side bench (perf + `check_equiv` correctness). Kernels under `metaltile-std/src/mlx/` only.
- `tile test` — GPU correctness sweep, no perf. Faster than `bench`.
- `tile profile --kernel <name>` — per-kernel profiling. Reach for this when a kernel regresses.
- `tile inspect --kernel <name>` — print IR + emitted MSL. Reach for this when debugging codegen.
- `tile diff` — compare emitted MSL across branches.

### Rust DSL philosophy

- **Improve the compiler, don't hand-write MSL.** When the DSL can't express a kernel pattern, extend the codegen instead of bypassing it.
- **Generics + macros for combinatorial variants** (bit widths, dtypes, group sizes). One generic `<T>` kernel beats five precision-specific copies. One `macro_rules!` body beats five hand-copied bit-width variants. Audit existing files for collapse opportunities whenever you touch them.

  **Reference example:** [`thewafflehaus/metaltile#3`](https://github.com/thewafflehaus/metaltile/pull/3) consolidated five `ffai/` files from per-bit-width copies to shared macro bodies — removed ~980 lines, IR + MSL identical.

  **Do this:**
  ```rust
  macro_rules! dequant_gather_body {
      ($bits:literal) => { /* one formula, bits as a literal */ };
  }
  #[kernel] pub fn dequant_gather_int3<T>(...) { dequant_gather_body!(3); }
  #[kernel] pub fn dequant_gather_int4<T>(...) { dequant_gather_body!(4); }
  #[kernel] pub fn dequant_gather_int8<T>(...) { dequant_gather_body!(8); }
  ```

  **Not this:**
  ```rust
  #[kernel] pub fn dequant_gather_int3<T>(...) { /* 60 lines hand-rolled int3 */ }
  #[kernel] pub fn dequant_gather_int4<T>(...) { /* 60 lines hand-rolled int4 */ }
  // …three more copies…
  ```

### Porting kernels from our `ekryski/mlx` fork

Custom kernels (turbo / gated-delta / SSM / etc.) live on the **`alpha`** branch of our MLX fork: [`ekryski/mlx@alpha`](https://github.com/ekryski/mlx/tree/alpha) (local checkout at `~/Development/personal/ai/mlx`, branch `alpha`). The relevant `.metal` sources are under `mlx/backend/metal/kernels/`.

1. Try existing metaltile DSL primitives + `#[kernel]` macros first.
2. Missing primitive? Extend the DSL (body parser + IR + MSL codegen). Don't write raw MSL.
3. New AURA / GDN / SSM / sinks / sliding-window / vision / audio kernels start under `crates/metaltile-std/src/ffai/`. They graduate to `mlx/` once an MLX counterpart exists at the pinned `MLX_COMMIT`.
4. Switch metaltile's `MLX_COMMIT` to `ekryski/mlx@alpha` so `tile bench`'s side-by-side runner has a reference for our custom kernels. After each port, compare the metaltile-generated MSL against the corresponding kernel on the `alpha` branch.

### Testing kernels

- **MLX side-by-side via `tile bench`** is the correctness oracle whenever a counterpart exists at the pinned commit (our fork or upstream). Match within `check_equiv` tolerance.
- **No MLX counterpart?** Validate via an FFAI integration test against a real model — coherent output on a fixed prompt + seed. Once green, backport the verified shapes (`head_dim`, `n_kv_heads`, `group_size`, `seq_len`, etc.) into metaltile as a `tile bench` row so future regressions are caught.
